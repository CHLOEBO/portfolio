De Morgan's Law

When there is an "!" before an if statement comparing two values, the "!" is distributed.
In addition, De Morgan's Law states that if this happens, the sign comparing the two values will change. For example, && becomes ||.
--------
E.x.
!(x && y) == !x || !y
--------
Short circut evaluation makes comparing two values more efficient. If using an "&&" to compare values and the first value is false, the system will automatically deem the statement as false. This saves time as the system does not need to check whether the other value is true or false.

Logic

An "if statement" checks whether a conditional is true and runs the code inside the statement if it is true. If the conditional is false, the code does not run.
--------
An "else if statement" follows an if statement and only runs if the preceding statement is false. If this statement is true the code inside of the statement runs.
--------
An "else statement" runs after any "if"/"if else" statements and will only run if any preceding if the conditionals of the preceding "if statements" are seemed false.
--------
To compare values, there are multiple different operators. The first is "&&" which means "and". Both values being compared using this operator must be true for the statement to be evaluated as true. Next, is "or" which is represented in code as "||". With an "||" operator, if either or both of the values being compared are true, the statement will be true. Finally, to see if the value is not equal to another value, use "!=". The "!" means not, therefore it will check if the value is not equal to the one it's being compared to.

== v.s. .equals

While "==" and ".equals" both check if two values are equal, they are still different. When checking if the memory location is the same for two objects, "==" should be used. ".equals" would be used when checking if the values of the objects are the same.
--------
Example:
String new = "apple";
String old = "apple";
return 3 == 3;
return new == old;
return new.equals(old);
--------
These statements would return:
true
false
true
